# Style Matching

Character anchors should match the visual lane of the project they support.

## Rules

1. Put the render lane in the anchor prompt, not only in the project brief.
2. Keep character style language consistent with the intended provider output.
3. Prefer one dominant visual direction over mixed incompatible styles.

## Practical Mapping

| Project lane | Character style language |
|---|---|
| Cinematic live-action | photorealistic, cinematic lighting, filmic texture |
| Stylized UGC | clean product-lighting, creator-energy, social-video realism |
| Anime | anime linework, cel-shaded lighting, illustrative detail |
| Painterly or fantasy | painterly texture, storybook rendering, stylized brushwork |
| Neon noir | neon-noir grading, rim light, moody contrast, cinematic realism |

## What To Avoid

- "realistic anime cinematic Pixar" unless the hybridity is deliberate
- style language that changes between character anchors for the same cast
- project-wide grading cues that never make it into the anchor prompt

## Patch Guidance

When a character drifts, patch the smallest useful thing:

1. add missing style language
2. tighten the archetype lock
3. clarify the distinguishing accessory or silhouette

Recreate only after patching fails.
